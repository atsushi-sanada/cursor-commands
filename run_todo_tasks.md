# Todoタスク実行プロンプト

## 基本方針
- ユーザーからの指示を**実行可能な最小単位のTodoタスクに分解**して実行する
- **全てのTodoが完了するまで**、順次タスクを実行し続ける
- 各タスクの完了時に進捗を報告する

## Todo作成ルール

### 粒度の定義
- **1つのTodoは1つの具体的な作業**を表す
- 1つのTodoは**5〜30分程度で完了可能**なサイズに分解する
- 複数のステップが必要な場合は、それぞれを別のTodoとして作成する

### Todo作成時の必須項目
- **明確な作業内容**：何を実行するかが明確であること
- **完了条件**：どの状態になったら完了かが明確であること
- **依存関係**：他のTodoに依存する場合は明示する

## 実行手順

### 1. タスク分解フェーズ
1. ユーザーからの指示を分析し、必要な作業を洗い出す
2. 各作業を実行可能な最小単位のTodoに分解する
3. 依存関係を考慮して優先順位を設定する
4. `todo_write`ツールを使用してTodoリストを作成する

### 2. 実行フェーズ
1. **優先順位の高いTodoから順次実行**する
2. 各Todoの実行前に、必要な情報やファイルを確認する
3. Todoを実行し、結果を確認する
4. Todoが完了したら、`todo_write`ツールでステータスを`completed`に更新する
5. 次のTodoに進む

### 3. 完了確認フェーズ
1. 全てのTodoが`completed`状態になったことを確認する
2. 最終的な成果物が期待通りに完成しているか確認する
3. ユーザーに完了を報告する

## 進捗報告ルール

### 各Todo完了時
- Todoの完了を報告する
- 必要に応じて、実行結果の概要を簡潔に説明する

### エラー発生時
- エラーの内容を明確に報告する
- エラー解決のためのTodoを追加するか、既存のTodoを修正する
- エラー解決後、該当Todoを再実行する

## 中断・再開ルール

### 中断が必要な場合
- ユーザーからの明示的な指示がある場合のみ中断する
- 中断時は、現在の進捗状況と残りのTodoを報告する

### 再開時
- 既存のTodoリストを確認する
- 未完了のTodoから順次実行を再開する

## 品質チェックリスト

- [ ] 全ての指示がTodoに分解されているか
- [ ] 各Todoの粒度が適切か（5〜30分で完了可能か）
- [ ] 依存関係が明確になっているか
- [ ] 全てのTodoが完了しているか
- [ ] 最終成果物が期待通りに完成しているか
