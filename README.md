# cursor-commands

Cursor エディタで使用するコマンド/プロンプト集（Docs）を管理するリポジトリです。各ファイルは Cursor の Docs 機能で読み込んで使用します。

---

## 概要

このプロジェクトは、Cursor エディタでの開発作業を効率化するための**指示書（Docs）**を集約しています。バグ修正、テスト実行、Git 操作、ドキュメント生成など、よく使う作業を標準化されたプロンプトとして提供します。

---

## セットアップ

1. このリポジトリをクローンまたはダウンロードします
2. Cursor エディタでプロジェクトを開きます
3. 必要なコマンドファイルを Cursor の Docs として読み込みます

---

## コマンド一覧

### プロジェクトセットアップ系

| コマンド                | 内容                                                                                                                                                                                                            |
| ----------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `workflow-start.md`     | タスク実行の 5 段階フローを実行。要件定義 → 設計 → 実装計画 → issue 作成 → 実行の順で進める。アジャイル・スクラム開発を前提とし、段階的にタスクを実行する際に使用。                                             |
| `project-rule-setup.md` | 新規プロジェクトのセットアップを 3 段階で実行。AGENTS.md 作成、.cursor/rules/の提案・作成を行う。プロジェクトの思想と理念を定義し、それに基づいてルールを作成する。新規プロジェクトの初期セットアップ時に使用。 |

---

### タスク管理系

| コマンド            | 内容                                                                                                                                      |
| ------------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| `run-todo-tasks.md` | ユーザーからの指示を実行可能な最小単位の Todo タスクに分解して実行。粒度は 5〜30 分で完了可能。複雑な作業を段階的に実行したい場合に使用。 |
| `CreateTaskMd.md`   | `.cursor/tasks/task.md` ファイルを作成。タスク名、概要、作業内容、完了条件、関連情報、目安時間を含むテンプレートを提供。                  |

---

### バグ対応系

| コマンド        | 内容                                                                                                                                                                        |
| --------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `AnalyzeBug.md` | バグの原因を分析し、修正方法を提案（コード修正は行わない）。思考ロジック（問題 → 原因 → 本質 → 対策）に従って分析。バグが発生した際に原因分析と対策案を求めたい場合に使用。 |
| `FixBug.md`     | バグ修正を標準プロセスに従って実行。6 ステップ（本質確認 → 全体把握 → 設計 → ファクトチェック → 実施 → 報告）で実行。バグ修正を実行する際に使用。                           |
| `SetLogs.md`    | バグ発生箇所の切り分けが容易になるログ出力処理を挿入。ログ分類（[INIT], [INPUT], [PROCESS], [OUTPUT], [WARN], [ERROR], [DONE]）を使用。バグの再現性を高めたい場合に使用。   |

---

### Git 操作系

| コマンド                 | 内容                                                                                                                                                                                                                                   |
| ------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `CreateCommitComment.md` | Git コミットメッセージを作成。短く簡潔、箇条書き、タイトルと本文を分離した形式。Git コミット時に使用。                                                                                                                                 |
| `GitCommitPush.md`       | Git のコミットとプッシュを実行。変更内容確認 → ステージング → コミットメッセージ作成 → コミット実行 → プッシュ実行の流れ。`CreateCommitComment.md`のルールに従ってコミットメッセージを作成。変更をコミット・プッシュしたい場合に使用。 |

### テスト・品質管理系

| コマンド     | 内容                                                                                                                                                     |
| ------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `RunTest.md` | テストを実行し、成功するまで修正を繰り返す。リトライ時は`/AnalyzeBug`と`/FixBug`を使用。テスト成功後は README 更新も行う。テストを実行したい場合に使用。 |

### ドキュメント生成系

| コマンド                  | 内容                                                                                                                                                                                                                                                                                |
| ------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `CreateREADME.md`         | プロジェクト用の高品質な README.md を自動生成。日本語、中学生でも理解できる語り口、見出し最大 4 階層。章構成テンプレートと品質チェックリストを含む。新規プロジェクトの README を作成したい場合に使用。                                                                              |
| `DocumentationComment.md` | ソースコード内のすべての関数にドキュメンテーションコメントを自動付与。関数ごとに 5 関数ずつ自動処理。コメント形式は言語に応じて自動選択（C#: XML、JavaScript/TypeScript: JSDoc、Python: Docstring、Go: 通常コメント）。コードにドキュメンテーションコメントを追加したい場合に使用。 |
| `MarkdownFormatter.md`    | Markdown ドキュメントを整形。見出し階層ルール（H1 は 1 回のみ、H2→H3→H4 の順序）、コードブロックルール（フェンス付き、言語明示）、リンクルール（インラインリンク推奨）を適用。Markdown ファイルを整形したい場合に使用。                                                             |
| `CleanAiDocs.md`          | AI が生成した文章を資料として使えるように整形。AI の会話文・前段・後段を削除し、調査内容のみを残して Markdown で整形。会話調や口語表現を除去し、ビジネス資料として成立する文体に変換。AI の回答を資料として保存したい場合に使用。                                                   |

## 使い方

### 基本的な使い方

1. Cursor エディタでプロジェクトを開きます
2. 必要なコマンドファイルを Cursor の Docs として読み込みます
3. チャットでコマンド名を指定するか、ファイルの内容を参照して指示を出します

### 例: バグ修正を行う場合

1. `FixBug.md` を Docs として読み込みます
2. チャットで「FixBug.md の手順に従ってバグを修正してください」と指示します
3. AI が 6 ステップのプロセスに従ってバグ修正を実行します

### 例: README を作成する場合

1. `CreateREADME.md` を Docs として読み込みます
2. チャットで「CreateREADME.md のルールに従って README を作成してください」と指示します
3. AI がテンプレートに従って README を生成します

## トラブルシューティング

### コマンドが認識されない

**症状**: Cursor で Docs として読み込んでもコマンドが動作しない

**原因**: ファイルパスが正しく指定されていない、または Docs として正しく読み込まれていない

**解決手順**:

1. Cursor の Docs 機能でファイルを正しく読み込んでいるか確認
2. ファイルパスが正しいか確認
3. チャットでファイル名を明示的に指定して使用

---

### コマンドの実行結果が期待と異なる

**症状**: コマンドを実行したが、期待通りの結果が得られない

**原因**: コマンドの前提条件が満たされていない、または指示が不明確

**解決手順**:

1. 各コマンドファイルの「前提条件」や「使い方」セクションを確認
2. プロジェクトの状態がコマンドの前提条件を満たしているか確認
3. 必要に応じて、コマンドファイルの内容を確認して手動で調整

---

### 複数のコマンドを組み合わせて使いたい

**症状**: 複数のコマンドを順番に実行したい

**解決手順**:

1. `run-todo-tasks.md` を使用してタスクを分解
2. 各タスクで必要なコマンドを順番に実行
3. または、チャットで複数のコマンドを順番に指定
