# cursor-commands

Cursor エディタで使用するコマンド集を管理するリポジトリです。

---

## 概要

Cursor エディタでの開発作業を効率化するための **コマンド（/コマンド）** を集約しています。
バグ修正、テスト実行、Git 操作、ドキュメント生成などの頻出作業を、再現性の高い手順として標準化します。

### このツールについて

- **できること（1機能1行）**
  - バグ修正、テスト実行、Git 操作、ドキュメント整形などの作業を `/コマンド` で呼び出せるようにする
  - よく使う作業フローをプロンプトとして共通化し、誰が実行しても同じ手順に揃える

---

## クイックスタート

### 初回のみ（グローバルに使う：推奨）

1. `"%USERPROFILE%\.cursor\commands"` にこのリポジトリを配置します（Clone かコピー）
2. Cursor を再起動します
3. チャットで `/コマンド名` を入力して実行します

### 毎回

1. チャットで `/コマンド名` を入力します
2. 指示に従って作業を進めます

**例**: `/fix_bug` と入力してバグ修正を実行

---

## セットアップ

コマンドファイル(.md)を以下のいずれかのディレクトリに配置します：

- **グローバルに使用する場合（推奨）**: `"%USERPROFILE%\.cursor\commands"`
- **プロジェクト固有の場合**: `".cursor\commands"` 以下に配置

※ディレクトリに直接 Clone するとメンテナンスが楽です

配置後、Cursor エディタを再起動すると `/コマンド名` で呼び出せます。

### プロジェクトセットアップ系

| コマンド                  | 内容                                                                                       |
| ------------------------- | ------------------------------------------------------------------------------------------ |
| `start_workflow.md`       | 仕様駆動開発用。タスク実行の 5 段階フロー（要件定義 → 設計 → 実装計画 →issue 作成 → 実行） |
| `setup_project_rules.md`  | 新規プロジェクトのセットアップ（`AGENTS.md` を作成して、適切な Project Rule を作成）       |
| `setup_gas_environment.md`| GAS 開発環境構築（clasp 設定、プロジェクト構造、BAT ファイルの自動生成）                   |
| `create_vscode_settings.md`| プロジェクトルートに `.vscode` フォルダと `settings.json` を作成                          |
| `set_workspace_color.md`  | VSCode/Cursor のワークスペース色設定を `.vscode/settings.json` に追加または更新           |
| `fetch_project.md`        | 現在のプロジェクトについて把握する                                                       |

## その他

### トラブルシューティング

- **コマンドが認識されない**: コマンドが正しく配置されているか確認し、Cursor エディタを再起動してください
- **実行結果が期待と異なる**: 各コマンドファイルの前提条件を確認してください
- **複数コマンドを組み合わせる**: `run_todo_tasks.md` を使用してタスクを分解してください
