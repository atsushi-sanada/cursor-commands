# README 作成

以下の内容でREADMEを作成してください。

## 1. READMEの役割

* READMEは**入口説明書**
* 書くのは「概要」と「最短で使う方法」だけ
* **詳細仕様・例外・ロジックは禁止**（コードへ書く）

---

## 2. 必須セクション（順番固定）

```
概要
クイックスタート
セットアップ
使い方
その他
```

※ この5つ以外は書かない

---

## 3. READMEテンプレート

```markdown
# プロジェクト名

## 概要

プロジェクトの目的と、何ができるかを2〜4行で書く。

### このツールについて

- できること（1機能1行）
- できること（1機能1行）
- できること（1機能1行）

※ 重要な制約・注意がある場合のみ1行で書く

---

## クイックスタート

1. 手順
2. 手順
3. 手順

---

## セットアップ

- 前提条件

1. 手順
2. 手順
3. 手順

---

## 使い方

1. 手順
2. 手順
3. 手順

---

## その他

- TIPS
- オプション
- 詳細はコード内ドキュメント参照
```

---

## 4. セクション別ルール（最低限）

### 概要

* 2〜4行厳守
* 目的と成果を書く
* 抽象表現禁止

### クイックスタート

* 各3ステップ以内
* 実行方法を曖昧にしない

### セットアップ

* 前提条件を書く
* 手順は最大5ステップ
* 説明文は禁止

### 使い方

* クイックスタートに記載していない詳しい使い方
* 番号付きリスト

### その他

* 最低限のみ
* 膨らみ始めたら削除対象

---

## 5. 禁止事項

* READMEに仕様を書く
* READMEに例外処理を書く
* READMEで全部説明しようとする
* READMEが1画面で読めなくなる構成

---

## 6. 最終チェック（Yes / No）

* 何のツールか一瞬で分かるか
* コピペで動かせるか
* 詳細はコードに逃げているか

**Yesが3つ揃わなければ書き直し。**
