# .cursor/rules 提案リスト

## 概要

このドキュメントは、`AGENTS.md` の思想と方針に整合する形で **どの `.cursor/rules/*.mdc` を作成するか**の提案一覧です。

## 提案ルール一覧（.mdc）

| ルールファイル案             | 目的（何を揃えるか）                                                                                                                     | 主な適用範囲（globs 案）           | alwaysApply 案 |
| ---------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------- | -------------- |
| `ai-response-governance.mdc` | 推測で断定しない・不足情報は質問・全体像 → 詳細・段階進行など、AI の振る舞いを統一する（AGENTS: 2,3,4）                                  | `**/*`                             | `true`         |
| `commands-authoring.mdc`     | Commands（/コマンド）を「再現性の高い手順」として書くための構成・注意点（フェーズ確認、成果物定義、最小変更）を統一する（AGENTS: 1,2,3） | `*.md`, `.cursor/commands/**/*.md` | `true`         |
| `markdown-writing.mdc`       | Markdown の書式（見出し階層、コードフェンス、リンク表記など）を統一する（AGENTS: 2）                                                     | `**/*.md`                          | `true`         |
| `repo-naming-structure.mdc`  | 追加するファイル/フォルダの命名規則と配置方針（小文字+ハイフン、深いネスト回避等）を統一する（AGENTS: 2,3）                              | `**/*`                             | `true`         |
| `readme-guidelines.mdc`      | README を入口説明書として簡潔に保つ（概要/セットアップ/使い方/その他）方針を統一する（AGENTS: 2,3）                                      | `README.md`                        | `false`        |
| `git-commit-message.mdc`     | コミットメッセージ生成の形式（短く、タイトル+本文、箇条書き、日本語）を統一する（AGENTS: 2,3）                                           | `**/*`                             | `false`        |
| `bat-file-guidelines.mdc`    | `.bat/.cmd` を作る場合の文字コード・改行・pause 制御・コメント方式を統一する（AGENTS: 2,3）                                              | `**/*.bat`, `**/*.cmd`             | `false`        |

## 補足

- **目的**: ルールは「禁止を増やす」ためではなく、Commands を運用しやすくするために **迷いを減らす**ことが目的です。
- **適用範囲（globs）**: 実運用でズレが出たら、作成フェーズで微調整します。
