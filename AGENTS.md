# プロジェクトの基本方針（AGENTS.md）

## 1. プロジェクトの目的

- Cursor の公式仕様である Commands（/コマンド）を作成・管理し、開発作業を効率化する。
- よく使うプロンプトや作業フローをコマンドにまとめて頻出作業を「再現性の高い手順」として標準化する。
- 誰が使っても同じ品質の結果が得られるよう、指示の粒度と流れを統一する。

## 2. コンセプト・思想

- 誰が見ても理解できることを最優先にし、迷いのない指示と構造を維持する。
- 保守性と拡張性を重視し、追加・変更がしやすい最小構成を志向する。
- 推測に依存せず、根拠と検証可能性を重視して「再現できる手順」に落とし込む。
- 運用を前提に、引き継ぎや再利用が容易な一貫性を守る。

## 3. タスクの実行方針（判断の軸）

- 目的と期待結果を先に明確化し、全体像 → 詳細の順で進める。
- 作業は段階的に分割し、各段階で結果確認と次段階の合意を取る。
- Cursor の公式ドキュメントを参照して Commands の最新仕様を把握し、必要な差分を取り入れる。
- 仕様や前提が不足している場合は、推測で進めず質問して確定させる。
- 追加実装は必要最小限に留め、過剰な機能追加や複雑化を避ける。
- 変更は責務分離を意識し、読みやすさ・保守性・再利用性を損なわない形で行う。

## 4. AI のコミュニケーション方針

- 新卒でも分かるように、背景・目的から説明し、手順をステップに分解して案内する。
- 不明点は必ず確認し、確定情報と未確定情報を混同しない。
- 推測だけで断定せず、必要に応じて事実確認を行ってから提案する。
- 進捗を短く共有し、ユーザーが判断できるタイミングで確認を求める。

## 5. 適用範囲と遵守事項

- この AGENTS.md に記載された思想と方針は、AI の判断・説明・提案に常に適用される。
- .cursor/rules の具体ルールと矛盾する場合は、この方針に基づいて解決を試みる。
- AI は作業中、判断に迷った場合は必ず本ファイルの思想に立ち返る。
